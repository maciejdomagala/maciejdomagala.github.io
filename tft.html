Temporal Fusion Transformer - analysis

Posted on Sat, Aug 22, 2020 deep learning time series

Source paper: https://arxiv.org/pdf/1912.09363.pdf

Selected repos:

  • https://github.com/jdb78/pytorch-forecasting - pytorch lightning based implementation,
  • https://github.com/google-research/google-research/tree/master/tft - source code, tensorflow.

This is simple analysis of the method described in "Temporal Fusion Transformers for Interpretable Multi-horizon Time Series Forecasting" paper with application in financial indices prediction. Main goal is to understand the model structure, its strenghts, its limitations and overall implementation.

First look

Authors of the paper tested and proposed the solution for time-series prediction in several cases, mainly - traffic intensity, electricity usage, retail store prices and financial volatility. The architecture is quite novel, using Multi Head Attention mechanism and Variable Selection Networks as the core added functionalities. New interesting and important feature is using static features from the future simultaneously with standard backcast variables.

Application

The data in traffic and electricity datasets should prove easier to predict. Main reason for that should be clearly visible seasonality and the trend of time series. These types of series are quite well predictable in general. Hence, first attempts were to check whether the model is able to converge for this data.

Below is the sample of quantile prediction made for these datasets. The model was virtually not optimized for both datasets, the LSTM hidden size was a bit lower to decrease computation time.

As we can notice, the predictions are 'acceptable' for this state of the model. It only proves some basic prediction power of the architecture, exactly was we were striking for. Model seems to be calibrating to the data, the datasets are quite large so improving these results would require using some hyperparameter optimizer like Optuna.

One of the main functionalities of the architecture is its 'interpretability'. With this framework it mainly means we can plot the percentage of importance of a given variable in final prediction.

This seems to be quite intuitive for Traffic dataset, since hour of the day and target are the most valuable resources that model is using. That would agree with ones analysis of seasonal series as well. On top of that, we can also demand from the model a plot showing us the attention mechanism and where is the main focus for the time series when learning.

Prediction for S&P500 financial index

Data was acquired from the Yahoo Finance website. The dataset consist of data between 03/01/2000 and 16/09/2020, with a total of ~5000 timesteps. Variables available for usage:

  • date,
  • close price,
  • open price,
  • index volatility.

One of the architecture novelties is usage of future (known) data, such as day or month. It was straightforward to generate additional variables that could be useful for the analysis, such as:

  • day of the week,
  • day of the month,
  • week of the year,
  • time index.

Main idea for the model is prediction in horizon of 10 days (10 timesteps) given some arbitraty (later defined) long window of backcast data.

Training

Two repositories were used:

  • Tensorflow based code with the model definition from Google-research,
  • Pytorch based code.

I wanted to get familiar with both implementations.

Using the first, the 'psuedo out-of-sample' prediction was performed. Most importantly, it proved that data can be processed in a similar fashion that base datasets and is interpreted correctly.

Rest of the experiments was performed using Pytorch.

For the training, random sampling of windows of length 40 was performed. 30 of them were used as encoder input, 10 of them - length of the forecast.The dataset that was used was relatively small, so I decided to create a 80%/20% dataset partition to generate train and validation sets. The quality of the model was verified solely on the validation dataset.

For backpropagation, recommended quantile loss function was applied with 7 quantiles (0.02, 0.1, 0.25, 0.5, 0.75, 0.9, 0.98). Simple hyperparameter optimization was performed using Optuna, getting optimal learning_rate, attention_head_size and hidden_state_size. Model was trained for ~100 epochs until the loss hit the plateau.

Some realisations are clearly far away from series targets, even though they can be explained by drastic changes in series direction and confusing the model. Model is quite vulnerable to white-noise (small dataset sample) and is not fully optimized, which affects prediction.

Below is the analysis of variable importance with separation to features known and unknown in the future. Target variable, log volatility and open_to_close values seem to be the most important while creating a prediction. Week of year seems to be crucial for categorical values inclusion.

Additionally, the naive predictor was used to benchmark the results. It is defined as constant value throughout the future range length and is established by taking the last seen value (last encoder input). For comparison, we calculated the mean of the sums of average errors (Mean Average Error) for all of the samples from validation set. The model results were similar to the benchmark, with the standard deviation of 20-30 units.
