Hierarchical clustering with NLP application - problem-based study

Posted on Sat, Sep 12, 2020 machine learning NLP clustering

My analysis and research regarding the idea of finding comparable companies within a certain dataset of companies.

Problem statement

Assume that we have a dataset of 40,000 public companies with information such as Company Name, Description, SI code, Stock Price etc. Assume that we also have a list of 5000 sets of public companies that are considered to be comparable.

Task: create a solution that will be able to generate recommendations across all 40,000 public companies.

Understanding of the problem

Area of the problem

The challenge is described as 'designing an approach for a solution that is capable of providing recommendations of operationally comparable companies based solely on an arbitrary company description'. This can be translated to finding groups of companies that are considered to be operationally comparable. This sounds similar to the group of problems solved by clustering methods. From a large dataset of different companies we want to create many clusters that would contain comparable items. Usually, the clustering algorithms are used as unsupervised learning methods. In this case, we have some additional information, which is the already labelled (grouped) part of the data belonging to Company Sets. That would mean that we have additional resources to consider and we can think about this problem in the category of semi-supervised clustering. Also, given the way of final partitioning of the data, we can think about some hierarchical clustering methods that would enable us to go arbitrary deep with the grouping.

Handling of the data

It's important to notice that the dataset we have for our usage contains different types of data:

  • Company Description - text,
  • Standard Industry Code - can be treated as categorical feature, mostly when we take divisions/major groups into consideration,
  • Stock Price, Annual Revenue - numerical data of different range and general distribution.

This leads to a few different possible ways of handling the data:

  1. Converting the data so we can apply a solution that would take all categories into consideration in one model e.g. vectorizing the text using Bag-of-Words or TFIDF, handling categorical data with i.e. one-hot-encoding, performing standardization on the numeric data.
  2. Handling the problem with model for text data and model for non-text data.
  3. Treating the data separately for each variable and developing a way of combining the information in aggregated fashion.

All of the methods have certain pros and cons. It's important to note some observations:

  1. There is a difference in usefulness of the data in each category, i.e. text data seems to be a much better source of information than SIC code.
  2. Using first solution could potentially lead to loss of important data, most critically for the description. We might be able extract more information from text using other NLP kind of solutions with the text not being recklessly converted.
  3. Splitting the problem into several models could lead to some difficulties when it comes to combining the results.
  4. Task is to identify operationally comparable companies, which is defined by:
    • similarity of the area of business (products/services) - best described by text feature,
    • similarity in the scale - best described by numeric features.
  5. Some initial assumptions about the data correlation can be made. Text data should be correlated with the SIC code, numeric data informing about financial scale should be correlated.

Researching and developing ideas

Some reading prior to proper research

https://www.osha.gov/pls/imis/sic_manual.html - list of Standard Industry Classification codes with partition regarding 'divisions' and 'major groups'. It was mentioned in the task description that it might not be the best determinant of the company area of business. Indeed, Tesla, according to siccode database, is listed under 'Automotive Trimmings, Apparel Findings, and Related Products', which might not be the best fit. Even though, the SIC code seems to be providing also some more general information about the company, categorizing it as e.g. 'manufacturing' or 'services' type of company.

Latent dirichlet allocation

Original paper

It uses topic modelling - idea of representing the text as a mixture of topics and representing topics as a mixture of words. Overview of the model can be found here and here.

It seemed attractive from the perspective that each group of companies could be identified as a separate topic. The model itself doesn't look like a good fit for our problem though. The limitations include:

  • It works well for quite limited number of topics and lots of words in the text. Our problem is reversed, we have limited number of words but potentially hundreds of topics linked to respective groups.
  • It doesn't generalize on the type of text it did not see before - there would be little use from the already grouped data that we have.

There are some extensions of this model that could adjust a bit better to our needs, but overall this does not seem to be sufficient.

Doc2vec

This post seems to be related to this task. The problem statement is similar to our task - finding comparable companies based on a text description. Data used there was acquired via scraping 40,000 pages including information about business description, products etc. The length of each description is similar to our data, probably more after performing some preprocessing (removing stopwords, punctuation etc.).

Remarks:

  • Uses PV-DBOW (model from this paper). The choice seems appropriate, although there are other alternatives (i.e. PV-DM model) that could be used here.
  • Solution does not include clustering - it is left in the form of using cosine similarity to find connected companies.

It looks like some inspiration can be taken from that work. There is code available here and here for simple preprocessing using pandas and the model building. Would also prove some merit to get familiar with these notes, mostly to know more about CBOW and Skip-Gram models.

Clustering

Decided to go a bit deeper in this direction at first. It feels that having a priori information about some groups of companies is a factor that should be included for solution building.

This article about some semi-supervised clustering methods, including algorithms for partially labelled data and observations constraining methods looks promising.

Some papers that are somewhat addressing the problem statement:

  • https://arxiv.org/pdf/1806.01547.pdf - semi-supervised clustering with neural networks
  • https://arxiv.org/pdf/1602.06797.pdf - semi-supervised Clustering for Short Text via Deep Representation Learning
  • http://meta-synthesis.amss.cas.cn/Publication/MSKS_Publications/Journal_papers/PaperlistsJ/201703/P020170301726951517352.pdf - TESC: An approach to TExt classification using Semi-supervised Clustering

It is a good idea to focus solely on few points:

  • What datasets were used for modelling?
  • What type of information was processed in the pipeline?
  • Are there any sources of code available for solutions?

First paper was custom made for image type of data, which poorly translates to our task. Second paper tackled the problem of 'short text' clustering. It possibly could be expanded for our problem, used LSTM + CNN type of architecture. All of the papers were addressing the problem slightly different from ours, since the datasets were large with small number of clusters (mostly pre-defined). Found some of the code here for similar architectures.

It seems that clustering with the methods mentioned in above papers is not what we are looking for. Given the fact that we actually need a large number of groups with only several companies in each group makes it look more like a decomposition problem. Good path might be the hierarchical clustering algorithms family.

https://ieeexplore.ieee.org/document/8122166 - this paper takes the idea of cosine similarity and uses it for hierarchical clustering.

https://users.cs.fiu.edu/archive/taoli/pub/semi-supervised-2011.pdf - this describes the idea of semi-supervised hierarchical clustering used in text, ideas look interesting but it's difficult to find any code implementation of this.

It is a straightforward way of proceeding but with the cost of making some assumptions, for instance the soft type of clustering (like hdbscan) won't be an option.

Some 'technical' research and ideas of a solution

Assuming that we are able to achieve high accuracy of initial clustering, we can use the statistical models for numeric data to alter the final clusters. In a very non-creative manner, the second part of the grouping might involve calculating overall distribution of the numeric features and predefining the classes that we can use.

Idea:

  1. Using nltk and gensim packages for text processing. Would propose using mainly the latter - it has the basic functionalities such as removing stop-words, tokenization, stemming/lemmatization that we would like to use.
  2. Incorporating doc2vec implementation (again in gensim) to get the vector representations for each of the description. There is also a quite straightforward tutorial on how to apply each of the functions. This is the part that would need to be adjusted empirically - the continuous bow, distributed bow or distributed memory models could be checked. It is highly dependent on the data, same with other hyperparameters (e.g. length of the resulting vector).
  3. Performing a hierarchical clustering. This is the part for which I think some more research time could be very useful. I would perform the clustering on the text data that we have, although there are some solutions that are taking into account mixed-data-type clustering, which could be integrated here. Also none of the semi-supervised methods are ready to apply here, but I would not resign from that completely. For this step, I would most likely go for scikit-learn package. There is direct implementation of agglomerative clustering algorithm (that was mentioned in one of the papers above). That would be a better fit for the problem, since it clusters 'bottoms-up' - starts with singleton clusters and merges them taking a given metric into account. For us it could be cosine similarity, Jaccard similarity or other (to experiment).
  4. Based on the data that is already grouped, calculate the distribution of numeric and categorical classes connected to the 'scale' description of the company. Categorical data could be converted to match the normalized variables from numeric columns. According to the task, we have large enough statistical sample to approximate the distribution with high level of confidence, which makes it applicable to the new clusters. With that method, create another partition of the data.
  5. Use a clustering evaluation method for the already grouped sample (in a manner of supervised learning). That can be viewed as models and parameters tuning for accurate separation of the data. Again, scikit-learn provides some built-in metrics, such as Rand index. Some more types of evaluations can be custom made (https://nlp.stanford.edu/IR-book/html/htmledition/evaluation-of-clustering-1.html - this paper explains the usage of i.e. purity metric).
